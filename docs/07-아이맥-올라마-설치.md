# 아이맥(iMac M1)에 Ollama 설치 및 qwen2.5-coder:14b 운영하기

## 배경

아이맥 M1은 Apple Silicon(Metal GPU)이 탑재되어 있어 LLM 추론에 매우 적합한 환경입니다.
Ollama를 Lima VM 내부가 아닌 **macOS 호스트에 직접 설치**하면, Metal GPU 가속을 통해
14B 파라미터 모델도 수 초 내에 첫 토큰을 생성할 수 있습니다.

설치 후에는 K3s 클러스터 내 다른 파드가 `ollama.keun-federation.svc.cluster.local:11434`
주소를 통해 Ollama API에 접근할 수 있도록 Kubernetes Service/Endpoints를 구성합니다.

> 💡 **왜 macOS 호스트에 설치하나요?**
> Lima VM 내부에서 Ollama를 실행하면 Apple M1의 Metal GPU를 활용할 수 없습니다.
> macOS 위에서 직접 실행하면 메모리 대역폭과 GPU 가속을 모두 활용할 수 있습니다.

---

## 아키텍처

```
[K3s 파드] → ollama.keun-federation.svc.cluster.local:11434
                 ↓ (Endpoints)
          [iMac macOS 호스트 : 192.168.x.30:11434]
                 ↓
          ollama serve  (Metal GPU 가속)
                 ↓
          qwen2.5-coder:14b
```

| 항목 | 요구사항 |
|------|----------|

| Homebrew | 설치되어 있거나 스크립트가 자동 설치 |

## 1단계: 자동 설치 스크립트 실행

저장소에 포함된 스크립트가 아래 작업을 자동으로 수행합니다:

1. Homebrew 확인 및 설치
2. Ollama 설치 (`brew install ollama`)
3. launchd 서비스 등록 (로그인 시 자동 시작)
4. `qwen2.5-coder:14b` 모델 다운로드
5. 동작 확인 (smoke test)

```bash
# 저장소 루트에서 실행
chmod +x scripts/setup/ollama-imac-setup.sh
./scripts/setup/ollama-imac-setup.sh
```

### 고급 옵션 (환경 변수)

| 변수 | 기본값 | 설명 |
|------|--------|------|
| `OLLAMA_HOST` | `0.0.0.0:11434` | Ollama 서버 바인드 주소 |
| `MODEL` | `qwen2.5-coder:14b` | 설치할 모델 |

```bash
# 다른 모델을 설치하거나 localhost 전용으로 제한할 때
MODEL=qwen2.5-coder:7b ./scripts/setup/ollama-imac-setup.sh
OLLAMA_HOST=127.0.0.1:11434 ./scripts/setup/ollama-imac-setup.sh
```

## 3단계: K3s 클러스터에 Service 등록

Ollama가 macOS 호스트에서 실행되므로, Kubernetes Endpoints + Service를 생성해
클러스터 내 파드가 안정적인 DNS 이름으로 접근할 수 있게 합니다.

### iMac IP 확인

```bash
# macOS 터미널에서 실행 (이더넷 또는 Wi-Fi LAN IP)
ipconfig getifaddr en0    # 이더넷
ipconfig getifaddr en1    # Wi-Fi (해당 경우)
```

### 매니페스트 적용

```bash
# manifests/ollama/ollama-service.yaml의 OLLAMA_IMAC_IP를 실제 IP로 대체한 후 적용
sed 's/OLLAMA_IMAC_IP/192.168.x.30/g' manifests/ollama/ollama-service.yaml \
  | kubectl apply -f -

# 또는 파일을 직접 편집하고 적용
kubectl apply -f manifests/ollama/ollama-service.yaml
```

### 클러스터 내 연결 확인

```bash
# 테스트 파드를 실행해 클러스터 내부에서 Ollama API 접근 확인
kubectl run ollama-test --rm -it --image=curlimages/curl --restart=Never \
  --namespace=keun-federation \
  -- curl -sf http://ollama.keun-federation.svc.cluster.local:11434/

# 응답 예시:
# Ollama is running
```

## launchd 서비스 관리

스크립트는 Ollama를 `~/Library/LaunchAgents/io.ollama.server.plist`로 등록합니다.
macOS 로그인 시 자동으로 시작되며, 아래 명령으로 관리할 수 있습니다:

```bash
# 서비스 상태 확인
launchctl list | grep ollama

# 서비스 중지
launchctl unload ~/Library/LaunchAgents/io.ollama.server.plist

# 서비스 시작
launchctl load -w ~/Library/LaunchAgents/io.ollama.server.plist

# 로그 확인
tail -f /tmp/ollama.log
tail -f /tmp/ollama.err
```

## 트러블슈팅

### Ollama가 시작되지 않는 경우

```bash
# 로그 확인
cat /tmp/ollama.err

# 포트 충돌 확인
lsof -i :11434

# 직접 실행해 오류 메시지 확인
OLLAMA_HOST=0.0.0.0:11434 ollama serve
```

### K3s 파드에서 Ollama에 연결되지 않는 경우

```bash
# 1. iMac 방화벽에서 11434 포트 허용 여부 확인
#    시스템 설정 → 네트워크 → 방화벽 → 수신 연결 허용에 Ollama 추가

# 2. Endpoints 리소스 확인
kubectl get endpoints ollama -n keun-federation
# 예상 출력:
# NAME     ENDPOINTS             AGE
# ollama   192.168.x.30:11434    1m

# 3. iMac에서 OLLAMA_HOST가 0.0.0.0으로 설정되어 있는지 확인
launchctl list io.ollama.server
cat ~/Library/LaunchAgents/io.ollama.server.plist | grep -A2 OLLAMA_HOST
```

### 메모리 부족 (14B 모델)

14B 모델은 약 9–10GB의 메모리가 필요합니다. iMac M1 16GB RAM에서 여유가 부족한 경우:

```bash
# 더 작은 모델로 대체
ollama pull qwen2.5-coder:7b    # ~4.7 GB
```

iMac M1에 Ollama가 Metal GPU 가속으로 실행되고,
K3s 클러스터 내 다른 파드에서 Ollama API에 접근할 수 있게 되었습니다.

```bash
# 마스터 노드에서 서비스 확인
kubectl get svc,endpoints -n keun-federation | grep ollama

# service/ollama    ClusterIP   10.43.x.x      <none>        11434/TCP
#
# NAME                ENDPOINTS
# endpoints/ollama    192.168.x.30:11434
```
