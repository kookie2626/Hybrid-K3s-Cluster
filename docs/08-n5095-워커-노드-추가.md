# N5095 미니PC 워커 노드 추가

## 배경

기존 클러스터(N100 마스터, 맥북프로 2014 워커, iMac Lima VM 워커)에 **Intel N5095 미니PC**를 추가 워커 노드로 편입했습니다.
N5095는 4코어·8GB RAM 구성으로, 저전력이면서도 안정적인 쿠버네티스 워크로드 처리가 가능합니다.

---

## 하드웨어 사양

| 항목 | 사양 |
|------|------|
| CPU | Intel Celeron N5095 (4코어, 최대 2.9GHz) |
| RAM | 8GB DDR4 |
| Storage | 256GB SSD |
| 네트워크 | GbE |
| 아키텍처 | x86-64 → Ubuntu 설치 가능 |

---

## Ubuntu Server 설치

### 1. Ubuntu Server 22.04 LTS ISO 다운로드

[Ubuntu 공식 사이트](https://ubuntu.com/download/server)에서 Ubuntu Server 22.04 LTS ISO를 다운로드합니다.

### 2. 부팅 USB 만들기

```bash
# macOS/Linux에서 dd 명령어로 부팅 USB 제작 (디바이스명은 환경에 따라 다름)
sudo dd if=ubuntu-22.04-live-server-amd64.iso of=/dev/sdX bs=4M status=progress
```

Windows에서는 [Rufus](https://rufus.ie/)를 사용해 부팅 USB를 만들 수 있습니다.

### 3. Ubuntu Server 설치

- N5095 미니PC에 USB를 꽂고 부팅합니다.
- BIOS에서 USB 부팅 우선순위를 설정합니다.
- Ubuntu Server 설치 마법사를 따라 진행합니다.
  - 언어: English
  - 키보드: Korean 또는 English US
  - 네트워크: DHCP 자동 설정 후 나중에 고정 IP로 변경
  - 스토리지: 전체 디스크 사용
  - 사용자명/호스트명/비밀번호 설정
  - **OpenSSH Server 설치 체크 (중요)**

설치 완료 후 USB를 제거하고 재부팅합니다.

## 고정 IP 설정

서버는 고정 IP를 사용해야 안정적으로 접속할 수 있습니다.

```bash
# 현재 인터페이스명 확인
ip a
```

```bash
# Netplan 설정 파일 작성
sudo nano /etc/netplan/01-keun-cluster.yaml
```

```yaml
network:
  version: 2
  ethernets:
    enp2s0:                  # 인터페이스명은 환경에 따라 다름 (ip a 로 확인)
      dhcp4: false
      addresses:
        - 192.168.x.40/24   # 원하는 고정 IP
      routes:
        - to: default
          via: 192.168.x.1  # 게이트웨이 (공유기 IP)
      nameservers:
        addresses:
          - 8.8.8.8
          - 1.1.1.1
```

```bash
sudo chmod 600 /etc/netplan/01-keun-cluster.yaml
sudo netplan apply
```

## K3s 에이전트 설치 및 클러스터 합류

### 1. K3s 커널 파라미터 설정

```bash
# 스왑 비활성화 (K3s 필수 요건)
sudo swapoff -a
sudo sed -i '/\sswap\s/d' /etc/fstab

# 커널 파라미터 설정
cat <<EOF | sudo tee /etc/sysctl.d/99-k3s.conf
net.bridge.bridge-nf-call-iptables  = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward                 = 1
EOF
sudo sysctl --system
```

### 2. 마스터 노드에서 노드 토큰 확인

N100 마스터 노드에서 다음 명령어로 토큰을 조회합니다:

```bash
sudo cat /var/lib/rancher/k3s/server/node-token
```

### 3. K3s 에이전트 설치

N5095에서 다음 명령어를 실행해 K3s 에이전트를 설치하고 클러스터에 합류합니다:

```bash
curl -sfL https://get.k3s.io | \
  K3S_URL=https://<N100-마스터-IP>:6443 \
  K3S_TOKEN=<노드-토큰> \
  sh -
```

### 4. 에이전트 서비스 상태 확인

```bash
sudo systemctl status k3s-agent
```

## 방화벽(UFW) 설정

```bash
sudo ufw allow OpenSSH
# K3s 에이전트가 마스터와 통신하는 데 필요한 포트
sudo ufw allow 8472/udp   # Flannel VXLAN
sudo ufw allow 10250/tcp  # Kubelet metrics
sudo ufw enable
sudo ufw status
```

## 결과 확인

마스터 노드(N100)에서 클러스터 노드 목록을 확인합니다:

```bash
kubectl get nodes
# NAME             STATUS   ROLES                  AGE
# n100             Ready    control-plane,master   --
# mbp-2014         Ready    <none>                 --
# lima-worker      Ready    <none>                 --
# n5095-worker     Ready    <none>                 --
```

`n5095-worker` 노드가 `Ready` 상태로 표시되면 클러스터 합류가 완료된 것입니다.

## 자동화 스크립트

설치 과정을 자동화하려면 다음 스크립트를 활용하세요:

```bash
# 스크립트 실행 (저장소 루트에서)
chmod +x scripts/setup/n5095-ubuntu-setup.sh

# 환경 변수로 마스터 IP와 토큰을 전달
export MASTER_IP=192.168.x.10
export NODE_TOKEN=<노드-토큰>
sudo -E scripts/setup/n5095-ubuntu-setup.sh
```
