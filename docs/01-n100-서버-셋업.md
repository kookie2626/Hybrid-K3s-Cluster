# N100 미니PC를 서버로 만들기

## 배경

쿠버네티스 클러스터를 구축하기에 앞서, Intel N100 미니PC를 홈서버로 세팅하는 것부터 시작했습니다.
N100은 저전력·소형 폼팩터임에도 불구하고 4코어·16GB RAM 구성이 가능해 홈랩 용도로 적합합니다.

## 하드웨어 사양

| 항목 | 사양 |
|------|------|
| CPU | Intel N100 (4코어, 최대 3.4GHz) |
| RAM | 16GB DDR5 |
| Storage | 256GB NVMe SSD |
| 네트워크 | 2.5GbE |

## Ubuntu Server 설치

### 1. Ubuntu Server 22.04 LTS ISO 다운로드

[Ubuntu 공식 사이트](https://ubuntu.com/download/server)에서 Ubuntu Server 22.04 LTS ISO를 다운로드합니다.

### 2. 부팅 USB 만들기

[Rufus](https://rufus.ie/) (Windows) 또는 `dd` 명령어 (macOS/Linux)로 부팅 USB를 제작합니다.

```bash
# macOS/Linux 예시 (디바이스명은 환경에 따라 다름)
sudo dd if=ubuntu-22.04-live-server-amd64.iso of=/dev/sdX bs=4M status=progress
```

### 3. Ubuntu Server 설치

- N100 미니PC에 USB를 꽂고 부팅합니다.
- BIOS에서 USB 부팅 우선순위를 설정합니다.
- Ubuntu Server 설치 마법사를 따라 진행합니다.
  - 언어: English
  - 키보드: Korean (또는 English US)
  - 네트워크: DHCP 자동 설정 후 나중에 고정 IP로 변경
  - 스토리지: 전체 디스크 사용
  - 사용자명/호스트명/비밀번호 설정
  - **OpenSSH Server 설치 체크 (중요)**

## 고정 IP 설정

서버는 고정 IP를 사용해야 안정적으로 접속할 수 있습니다.

```bash
# Netplan 설정 파일 편집
sudo nano /etc/netplan/00-installer-config.yaml
```

```yaml
network:
  version: 2
  ethernets:
    enp2s0:                  # 인터페이스명은 환경에 따라 다름
      dhcp4: false
      addresses:
        - 192.168.0.100/24   # 원하는 고정 IP
      routes:
        - to: default
          via: 192.168.0.1   # 게이트웨이 (공유기 IP)
      nameservers:
        addresses:
          - 8.8.8.8
          - 1.1.1.1
```

```bash
# 설정 적용
sudo netplan apply
```

## SSH 접속 설정

### 공개키 인증 설정 (비밀번호 대신 키 사용)

로컬 PC에서 SSH 키를 생성하고 서버에 등록합니다.

```bash
# 로컬 PC에서 SSH 키 생성
ssh-keygen -t ed25519 -C "n100-server"

# 공개키를 서버에 복사
ssh-copy-id username@192.168.0.100
```

### SSH 비밀번호 인증 비활성화 (보안 강화)

```bash
sudo nano /etc/ssh/sshd_config
```

아래 항목을 수정합니다:

```
PasswordAuthentication no
PubkeyAuthentication yes
```

```bash
sudo systemctl restart ssh
```

## 기본 패키지 업데이트 및 설치

```bash
# 패키지 목록 업데이트 및 업그레이드
sudo apt update && sudo apt upgrade -y

# 유용한 기본 패키지 설치
sudo apt install -y \
  curl \
  wget \
  git \
  vim \
  htop \
  btop \
  net-tools \
  unzip
```

## 자동 업데이트 설정

```bash
sudo apt install -y unattended-upgrades
sudo dpkg-reconfigure --priority=low unattended-upgrades
```

## 방화벽(UFW) 설정

```bash
# UFW 설치 및 SSH 허용
sudo ufw allow OpenSSH
sudo ufw enable
sudo ufw status
```

## 결과

이제 로컬 네트워크 어디서든 SSH로 N100 서버에 접속할 수 있습니다.

```bash
ssh username@192.168.0.100
```

이 서버를 기반으로 이후에 Docker를 설치하고, 최종적으로 K3s 쿠버네티스 클러스터의 노드로 활용하게 됩니다.
