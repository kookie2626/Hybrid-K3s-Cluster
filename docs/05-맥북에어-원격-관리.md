# 맥북에어에서 외부 원격으로 서버 관리하기

## 배경

클러스터를 구성한 뒤, 집 밖에서도 맥북에어로 서버를 편리하게 관리하고 싶었습니다.
집 내부 LAN에서는 IP를 직접 사용해 SSH나 kubectl로 접근할 수 있지만,
외부 네트워크(카페, 회사, 이동 중)에서는 공유기 NAT 뒤에 있는 서버에 곧바로 접근할 수 없습니다.

이 문제를 안정적으로 해결하기 위해 **각 장치에 고정 IP를 부여하고 공유기에서 포트포워딩을 설정**하는 방식을 선택했습니다.
VPN 서비스에 의존하지 않고, 순수하게 SSH와 네트워크 설정만으로 외부에서 안전하게 관리할 수 있습니다.

---

## 아키텍처 개요

```
[집 밖 / 외부 네트워크]                [공유기 포트포워딩]        [홈 LAN]

  맥북에어 (관리자)
  ssh -p 2210 user@<공인IP>  ──────►  공유기 WAN            N100 마스터   192.168.x.10
  ssh -p 2220 user@<공인IP>  ──────►  포트포워딩 규칙 →     MBP 2014 워커 192.168.x.20
  kubectl (API :6443)        ──────►  외부 6443 → 내부 6443
```

| 외부 포트 | 프로토콜 | 내부 장치          | 내부 포트 |
|-----------|----------|--------------------|-----------|
| 2210      | TCP      | N100 마스터 (192.168.x.10) | 22  |
| 2220      | TCP      | MBP 2014 워커 (192.168.x.20) | 22 |
| 6443      | TCP      | N100 마스터 (192.168.x.10) | 6443 |

> ⚠️ `192.168.x.10`에서 `x`는 실제 네트워크 세그먼트로 교체하세요 (예: 192.168.**1**.10). 공유기 DHCP 설정이나 `ip a` 명령으로 확인할 수 있습니다.

- 공인 IP는 ISP에서 발급하는 WAN IP입니다. DDNS가 없다면 IP 변경 시 업데이트가 필요합니다.
- 모든 통신은 SSH 암호화를 통해 보호됩니다.

---

## 사전 준비

1. 각 서버(N100, MBP 2014)에 고정 LAN IP가 설정되어 있을 것 (공유기 DHCP 예약 또는 netplan)
2. 공유기 관리 화면에서 포트포워딩 규칙 추가 권한
3. 현재 공인 IP 확인: `curl -s ifconfig.me`

---

## Step 1 — 서버에 SSH 원격 접속 설정

N100 마스터와 MBP 2014 워커 각각에서 아래 스크립트를 실행합니다.

```bash
chmod +x scripts/setup/ssh-port-forward-setup.sh
sudo scripts/setup/ssh-port-forward-setup.sh
```

스크립트가 자동으로:
- OpenSSH 서버를 설치하고 활성화합니다.
- 키 인증만 허용하고 비밀번호 인증 및 루트 로그인을 비활성화합니다.
- UFW에서 SSH(22)를 허용합니다.
- btop을 설치합니다 (시스템 리소스 모니터링용).

### 수동으로 설정하는 경우

```bash
# SSH 서버 설치 및 활성화
sudo apt install -y openssh-server
sudo systemctl enable --now ssh

# SSH 키 인증만 허용 (비밀번호 비활성화)
sudo nano /etc/ssh/sshd_config
# 아래 항목 수정:
#   PasswordAuthentication no
#   PermitRootLogin no
#   PubkeyAuthentication yes
sudo systemctl restart ssh

# btop 설치
sudo apt install -y btop

# UFW 방화벽에서 SSH 허용
sudo ufw allow OpenSSH
sudo ufw enable
```

---

## Step 2 — 공유기 포트포워딩 설정

공유기 관리 화면(일반적으로 `http://192.168.x.1`)에 접속하여 포트포워딩 규칙을 추가합니다.

| 규칙 이름      | 외부 포트 | 내부 IP         | 내부 포트 | 프로토콜 |
|----------------|-----------|-----------------|-----------|----------|
| n100-ssh       | 2210      | 192.168.x.10    | 22        | TCP      |
| mbp-2014-ssh   | 2220      | 192.168.x.20    | 22        | TCP      |
| k3s-api        | 6443      | 192.168.x.10    | 6443      | TCP      |

> 💡 공유기 모델마다 메뉴 위치가 다르지만 일반적으로 **NAT 설정** 또는 **포트포워딩** 메뉴에서 설정합니다.

---

## Step 3 — 맥북에어 설정

맥북에어에서 아래 스크립트를 실행합니다.

```bash
chmod +x scripts/setup/macbook-air-remote-setup.sh
./scripts/setup/macbook-air-remote-setup.sh
```

스크립트가 자동으로:
1. Homebrew 설치 (없는 경우)
2. `kubectl`, `helm` 설치
3. SSH 키 쌍 생성 (`~/.ssh/id_ed25519_k3s_cluster`)
4. `~/.ssh/config`에 서버 접속 단축 설정 추가
5. 마스터 노드에서 kubeconfig를 가져와 공인 IP로 패치

---

### 수동 설정 (스크립트 없이)

#### kubectl 설치

```bash
brew install kubectl
```

#### SSH 키 생성

```bash
ssh-keygen -t ed25519 -C "macbook-air-remote-mgmt" -f ~/.ssh/id_ed25519_k3s_cluster
# 공개키를 각 서버에 등록 (LAN에서 접속해 등록)
ssh-copy-id -i ~/.ssh/id_ed25519_k3s_cluster.pub <user>@192.168.x.10   # N100
ssh-copy-id -i ~/.ssh/id_ed25519_k3s_cluster.pub <user>@192.168.x.20   # MBP
```

#### ~/.ssh/config 설정

```
Host n100-master
    HostName <공인IP>             # 현재 공인 IP (curl -s ifconfig.me)
    Port 2210                    # 공유기에서 N100으로 포워딩한 외부 포트
    User <username>
    IdentityFile ~/.ssh/id_ed25519_k3s_cluster
    ServerAliveInterval 60
    ServerAliveCountMax 3

Host mbp-2014-worker
    HostName <공인IP>             # 같은 공인 IP, 다른 외부 포트
    Port 2220                    # 공유기에서 MBP로 포워딩한 외부 포트
    User <username>
    IdentityFile ~/.ssh/id_ed25519_k3s_cluster
    ServerAliveInterval 60
    ServerAliveCountMax 3
```

이제 다음 한 줄로 서버에 접속할 수 있습니다:

```bash
ssh n100-master
ssh mbp-2014-worker
```

#### kubeconfig 설정 (원격 kubectl 사용)

```bash
# 마스터 노드에서 kubeconfig 가져오기 (포트 2210 경유)
ssh n100-master "sudo cat /etc/rancher/k3s/k3s.yaml" > ~/.kube/config

# server 주소를 공인 IP로 교체
sed -i '' 's|https://127.0.0.1:6443|https://<공인IP>:6443|g' ~/.kube/config

# 권한 설정
chmod 600 ~/.kube/config
```

## Step 4 — 접속 확인

### SSH 접속 테스트

```bash
# 단축 이름으로 접속
ssh n100-master
ssh mbp-2014-worker
```

### btop으로 서버 리소스 모니터링

```bash
# SSH로 접속한 뒤 btop 실행
ssh n100-master "btop"
ssh mbp-2014-worker "btop"
```

### kubectl 원격 제어 테스트

```bash
# 노드 상태 확인
kubectl get nodes

# 예상 출력:
# NAME            STATUS   ROLES                  AGE
# n100            Ready    control-plane,master   7d
# mbp-2014        Ready    <none>                 5d
# lima-k3s-worker Ready    <none>                 3d

# 전체 파드 목록
kubectl get pods -A

# 클러스터 정보
kubectl cluster-info
```

## 일상적인 원격 관리

### 서버 상태 모니터링

```bash
# 노드 리소스 사용량
kubectl top nodes

# 파드 리소스 사용량
kubectl top pods -A

# 특정 파드 로그 확인
kubectl logs <pod-name> -n <namespace>

# 서버 시스템 상태 (SSH로 직접 접속)
ssh n100-master "btop"
ssh n100-master "df -h && free -h"
```

### 매니페스트 원격 배포

```bash
# 로컬 맥북에어에서 직접 배포
kubectl apply -f manifests/namespace.yaml
kubectl apply -f manifests/nginx-deployment.yaml
kubectl apply -f manifests/monitoring/
```

### 원격 포트 포워딩 (로컬에서 클러스터 서비스 접근)

```bash
# 클러스터 내 서비스를 맥북에어 로컬 포트로 연결
kubectl port-forward svc/<service-name> 8080:80 -n <namespace>
# 브라우저에서 http://localhost:8080 접근
```

## 보안 고려사항

| 항목 | 조치 |
|------|------|
| SSH 키 인증 | 비밀번호 인증 비활성화, 키 인증만 허용 |
| 루트 로그인 | `PermitRootLogin no` 설정 |
| kubeconfig 권한 | `chmod 600 ~/.kube/config` |
| K3s API 방화벽 | UFW에서 6443 포트는 신뢰할 수 있는 IP만 허용 (선택) |
| 공인 IP 변경 | ISP에서 동적 IP를 부여하는 경우 DDNS 서비스 사용 고려 |

### K3s API 포트 접근 제한 (선택)

특정 IP(예: 맥북에어 위치의 공인 IP)에서만 6443 포트를 허용할 수 있습니다:

```bash
# 특정 IP에서만 K3s API 허용
sudo ufw allow from <맥북에어-공인IP> to any port 6443
sudo ufw deny 6443
sudo ufw reload
```

## 문제 해결

### SSH 연결이 안 될 때

```bash
# 공인 IP 확인 (N100 서버에서)
curl -s ifconfig.me

# 포트포워딩 동작 확인 (맥북에어에서)
# nc가 없는 경우: brew install netcat 또는 아래 curl 대안 사용
nc -zv <공인IP> 2210   # N100 SSH 포트
nc -zv <공인IP> 6443   # K3s API 포트
# curl 대안:
# curl -s --connect-timeout 3 telnet://<공인IP>:2210 && echo "open" || echo "closed"

# SSH 디버그 모드로 연결 시도
ssh -v -p 2210 <user>@<공인IP>
```

### kubectl 연결 실패 시

```bash
# 마스터 노드 연결 테스트
curl -k https://<공인IP>:6443/healthz

# kubeconfig server 주소 확인
grep server ~/.kube/config
```

### kubeconfig 재발급이 필요한 경우

```bash
ssh n100-master "sudo cat /etc/rancher/k3s/k3s.yaml" \
  | sed 's|https://127.0.0.1:6443|https://<공인IP>:6443|g' \
  > ~/.kube/config
chmod 600 ~/.kube/config
```

## 결과

이제 맥북에어에서 집 밖 어디서든 다음과 같이 클러스터를 관리할 수 있습니다.

```bash
# 단 한 줄로 전체 클러스터 상태 확인
kubectl get nodes,pods -A

# SSH로 마스터 서버 직접 접속
ssh n100-master

# btop으로 서버 리소스 모니터링
ssh n100-master "btop"
```

고정 IP와 포트포워딩 설정 덕분에 별도의 VPN 서비스 없이도,
암호화된 SSH를 통해 어디서나 동일한 방법으로 서버를 안정적으로 관리할 수 있게 되었습니다.
