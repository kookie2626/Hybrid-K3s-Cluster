# 맥북에어에서 외부 원격으로 서버 관리하기

## 배경

클러스터를 구성한 뒤, 집 밖에서도 맥북에어로 서버를 편리하게 관리하고 싶었습니다.
집 내부 LAN에서는 IP를 직접 사용해 SSH나 kubectl로 접근할 수 있지만,
외부 네트워크(카페, 회사, 이동 중)에서는 공유기 NAT 뒤에 있는 서버에 곧바로 접근할 수 없습니다.

이 문제를 해결하기 위해 **Tailscale**을 선택했습니다.

> 💡 **Tailscale이란?**  
> WireGuard 기반의 제로-설정 VPN 서비스입니다. 방화벽 규칙이나 공유기 포트포워딩 없이도  
> 모든 장치를 하나의 가상 사설망으로 묶어주고, 각 장치에 고정된 Tailscale IP를 부여합니다.

---

## 아키텍처 개요

```
[집 밖 / 외부 네트워크]           [Tailscale 가상 네트워크]
                                   (100.x.x.x 대역)
  맥북에어 (관리자)
  Tailscale: 100.x.x.1  ──────►  N100 마스터     100.x.x.10
                                  MBP 2014 워커   100.x.x.20
                         (iMac M1 Lima VM은 iMac이 Tailscale에 참여하면 자동 연결)
```

- 모든 통신은 WireGuard로 암호화됩니다.
- 공유기 포트포워딩, DDNS 설정 불필요합니다.
- 집에 있을 때도 Tailscale IP를 그대로 사용하므로 장소에 무관하게 동일한 접속 방법을 유지합니다.

---

## 사전 준비

1. [Tailscale 계정 생성](https://tailscale.com/) (무료 플랜으로 최대 100대 장치 지원)
2. 각 서버(N100, MBP 2014)에 SSH로 접근 가능한 상태

---

## Step 1 — 서버에 Tailscale 설치

N100 마스터와 MBP 2014 워커 각각에서 아래 스크립트를 실행합니다.

```bash
chmod +x scripts/setup/tailscale-server-setup.sh
sudo scripts/setup/tailscale-server-setup.sh
```

스크립트가 자동으로:
- Tailscale 패키지를 설치합니다.
- `tailscaled` 서비스를 활성화합니다.
- `tailscale up --ssh` 명령으로 네트워크 참여 및 Tailscale SSH를 활성화합니다.
- 터미널에 출력되는 인증 URL을 브라우저에서 열어 로그인합니다.

### 수동으로 설치하는 경우

```bash
# Ubuntu 22.04
curl -fsSL https://tailscale.com/install.sh | sudo sh
sudo systemctl enable --now tailscaled
sudo tailscale up --ssh
```

인증 URL을 복사해 브라우저에서 열고, 로그인 후 "Authorize" 클릭합니다.

### Tailscale IP 확인

```bash
tailscale ip -4
# 예: 100.xx.xx.10
```

각 서버의 Tailscale IP를 기록해 둡니다. 이후 SSH 접속과 kubeconfig에서 사용합니다.

---

## Step 2 — 맥북에어 설정

맥북에어에서 아래 스크립트를 실행합니다.

```bash
chmod +x scripts/setup/macbook-air-remote-setup.sh
./scripts/setup/macbook-air-remote-setup.sh
```

스크립트가 자동으로:
1. Homebrew 설치 (없는 경우)
2. `kubectl`, `tailscale`, `helm` 설치
3. SSH 키 쌍 생성 (`~/.ssh/id_ed25519_k3s_cluster`)
4. `~/.ssh/config`에 서버 접속 단축 설정 추가
5. 마스터 노드에서 kubeconfig를 가져와 Tailscale IP로 패치

---

### 수동 설정 (스크립트 없이)

#### Tailscale 설치

```bash
# GUI 앱 (권장)
brew install --cask tailscale
open /Applications/Tailscale.app
# 앱을 열고 로그인 → 서버들과 같은 계정으로 로그인

# 또는 CLI만 필요한 경우
brew install tailscale
```

#### kubectl 설치

```bash
brew install kubectl
```

#### SSH 키 생성

```bash
ssh-keygen -t ed25519 -C "macbook-air-remote-mgmt" -f ~/.ssh/id_ed25519_k3s_cluster
# 공개키를 각 서버에 등록
ssh-copy-id -i ~/.ssh/id_ed25519_k3s_cluster.pub <user>@<tailscale-ip>
```

#### ~/.ssh/config 설정

```
Host n100-master
    HostName 100.xx.xx.10        # N100의 Tailscale IP
    User <username>
    IdentityFile ~/.ssh/id_ed25519_k3s_cluster
    ServerAliveInterval 60
    ServerAliveCountMax 3

Host mbp-2014-worker
    HostName 100.xx.xx.20        # MBP 2014의 Tailscale IP
    User <username>
    IdentityFile ~/.ssh/id_ed25519_k3s_cluster
    ServerAliveInterval 60
    ServerAliveCountMax 3
```

이제 다음 한 줄로 서버에 접속할 수 있습니다:

```bash
ssh n100-master
ssh mbp-2014-worker
```

#### kubeconfig 설정 (원격 kubectl 사용)

```bash
# 마스터 노드에서 kubeconfig 가져오기
ssh n100-master "sudo cat /etc/rancher/k3s/k3s.yaml" > ~/.kube/config

# server 주소를 Tailscale IP로 교체
sed -i '' 's|https://127.0.0.1:6443|https://100.xx.xx.10:6443|g' ~/.kube/config

# 권한 설정
chmod 600 ~/.kube/config
```

## Step 3 — 접속 확인

### SSH 접속 테스트

```bash
# 단축 이름으로 접속
ssh n100-master
ssh mbp-2014-worker
```

### kubectl 원격 제어 테스트

```bash
# 노드 상태 확인
kubectl get nodes

# 예상 출력:
# NAME            STATUS   ROLES                  AGE
# n100            Ready    control-plane,master   7d
# mbp-2014        Ready    <none>                 5d
# lima-k3s-worker Ready    <none>                 3d

# 전체 파드 목록
kubectl get pods -A

# 클러스터 정보
kubectl cluster-info
```

## 일상적인 원격 관리

### 서버 상태 모니터링

```bash
# 노드 리소스 사용량
kubectl top nodes

# 파드 리소스 사용량
kubectl top pods -A

# 특정 파드 로그 확인
kubectl logs <pod-name> -n <namespace>

# 서버 시스템 상태 (SSH로 직접 접속)
ssh n100-master "htop"
ssh n100-master "df -h && free -h"
```

### 매니페스트 원격 배포

```bash
# 로컬 맥북에어에서 직접 배포
kubectl apply -f manifests/namespace.yaml
kubectl apply -f manifests/nginx-deployment.yaml
kubectl apply -f manifests/monitoring/
```

### 원격 포트 포워딩 (로컬에서 클러스터 서비스 접근)

```bash
# 클러스터 내 서비스를 맥북에어 로컬 포트로 연결
kubectl port-forward svc/<service-name> 8080:80 -n <namespace>
# 브라우저에서 http://localhost:8080 접근
```

## 보안 고려사항

| 항목 | 조치 |
|------|------|
| Tailscale 인증 | 각 장치를 Tailscale 계정으로 명시적으로 인증 |
| SSH 키 인증 | 비밀번호 인증 비활성화, 키 인증만 허용 |
| kubeconfig 권한 | `chmod 600 ~/.kube/config` |
| K3s API 방화벽 | UFW에서 6443 포트는 Tailscale 인터페이스(`tailscale0`)만 허용 |
| Tailscale ACL | Tailscale Admin Console에서 ACL 정책으로 접근 제어 가능 |

### K3s API 포트를 Tailscale 전용으로 제한하기 (선택)

서버 내부 UFW에서 6443 포트를 Tailscale 네트워크 대역으로만 허용합니다:

```bash
# Tailscale IP 대역(100.64.0.0/10)에서만 6443 허용
sudo ufw allow in on tailscale0 to any port 6443
sudo ufw deny 6443
sudo ufw reload
```

## 문제 해결

### Tailscale 연결이 끊긴 경우

```bash
# 서버에서 (SSH로 접속 후)
sudo systemctl restart tailscaled
sudo tailscale up

# 맥북에어에서
tailscale up
```

### kubectl 연결 실패 시

```bash
# Tailscale 상태 확인
tailscale status

# 마스터 노드 연결 테스트
ping 100.xx.xx.10
curl -k https://100.xx.xx.10:6443/healthz

# kubeconfig server 주소 확인
grep server ~/.kube/config
```

### kubeconfig 재발급이 필요한 경우

```bash
ssh n100-master "sudo cat /etc/rancher/k3s/k3s.yaml" \
  | sed 's|https://127.0.0.1:6443|https://100.xx.xx.10:6443|g' \
  > ~/.kube/config
chmod 600 ~/.kube/config
```

## 결과

이제 맥북에어에서 집 밖 어디서든 다음과 같이 클러스터를 관리할 수 있습니다.

```bash
# 단 한 줄로 전체 클러스터 상태 확인
kubectl get nodes,pods -A

# SSH로 마스터 서버 직접 접속
ssh n100-master
```

Tailscale 덕분에 공유기 설정이나 DDNS 없이, 암호화된 안전한 터널을 통해  
어디서나 동일한 방법으로 서버를 관리할 수 있게 되었습니다.
